# CLI scenario suite (real deployed backend)

End-to-end **scenario** tests that drive the `wms` CLI against a **real, deployed,
isolated test backend** — not a mock and not local miniflare. They reset the test
database to a clean baseline, seed a medium 3PL volume from CSV, and assert realistic
warehouse flows through CLI output.

This complements `../integration/run.sh` (which boots a local `wrangler dev` and walks a
single lifecycle). Here the point is **volume + deterministic state against the live
deployed worker**.

## Target & safety

- Default target: **`erp-wms-backend-test.bizcard.workers.dev`** — the isolated
  `[env.test]` worker bound to a throwaway D1 (`wms_test`). It is the only environment
  where `POST /dev/reset` is enabled (`ALLOW_DESTRUCTIVE_RESET`). The suite **never**
  targets the real prod worker, and reset is a 404 there by design.

## Run

```bash
cli/tests/scenarios/run.sh
```

Requires `node` and `cargo`. Re-runnable (it resets first). Bootstrap developer
credentials are read from `backend/.test-env-secrets` (gitignored) or the
`WMS_BOOTSTRAP_EMAIL` / `WMS_BOOTSTRAP_PASSWORD` env vars.

### Config (env)

| Var | Default | Meaning |
|-----|---------|---------|
| `WMS_SCENARIO_ENDPOINT` | deployed test worker | backend origin |
| `WMS_BOOTSTRAP_EMAIL` / `WMS_BOOTSTRAP_PASSWORD` | from `.test-env-secrets` | bootstrap developer login |
| `TENANTS` | `ACME,GLOBEX` | shipper tenants to onboard |
| `N_LOC` / `N_ITEMS` / `N_STOCK` / `N_ORDERS` | `50 / 50 / 20 / 50` | volume knobs |

## What it covers

1. **Reset baseline** — `wms dev reset` wipes + re-seeds one developer (0 tenants, 1 user).
2. **Onboarding + catalog** — create tenants; bulk `location import` / `item import` (CSV).
3. **Inbound → stock** — receive + putaway across locations to build on-hand ("물류 확보"),
   asserting the **ledger conservation invariant** (`Σ inventory_ledger.delta == Σ inventory.qty_on_hand`).
4. **Bulk orders** — `order import` (auto-allocation) across both tenants.
5. **Order lifecycle** — a dedicated SKU: allocate → pick → pack → ship; on-hand drops; invariant still holds.
6. **Backorder** — demand beyond stock → `backorder`.
7. **Reports & alerts** — inventory report + low-stock alerts reflect the seeded data.
8. **Multi-tenant isolation** — one tenant's SKU is not visible under another tenant's scope.
9. **Authz boundary** — an unauthenticated write is rejected (exit 3).

Fixtures are generated by `gen-fixtures.mjs` to match the backend import grammars
(`items: sku,name,barcode,uom,min_level` · `locations: code,zone,type,capacity` ·
`orders: order_ref,sku,qty,ship_to_*`).
